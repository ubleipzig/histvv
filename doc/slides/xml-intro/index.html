XML

  • XML 1.0 (Februar 1998)
  • Vorbild SGML
  • "Ersatz" für HTML
  • textbasiert, menschen- und maschinenlesbar

SGML

Oxford English Dictionary 1985

Ein XML Beispiel

Krug, D. W. T., P. O., Geschichte d. alten Philosophie, n. s. Lehrbuche, 10 U. 4 T. öffentlich. 
<veranstaltung>
  <dozent><nachname>Krug</nachname>, <grad>D.</grad>
  <vorname>W. T.</vorname>, <funktion>P. O.</funktion></dozent>,
  <thema>Geschichte d. alten Philosophie, n. s. Lehrbuche</thema>,
  <zeit>10 U. 4 T.</zeit> <modus>öffentlich</modus>.
</veranstaltung>


=> Ein Text wird in seine semantischen Bestandteile aufgebrochen und durch Tags ausgezeichnet.

XML Grundlagen

  • Was ist ein Tag?
  • Was ist ein Element?
  • Was ist ein Attribut?
  • Baumstruktur

Tags

  • Start-Tag: <name>
  • End-Tag: </name>

Elemente

  • <dozent>...</dozent>
  • leeres Element: <hr/>

Attribute

  • <veranstaltung id="v4385">
  • "Adjektive" von XML
  • können nur in Start-Tags vorkommen

Sonstiges

  • Kommentare: <!-- lorem ipsum -->
  • Entity references: &amp; für "&",
  • Character references: &#169; für "©"
  • Namensräume 
    <html xmlns="http://www.w3.org/1999/xhtml">...</html>

XML: Baumstruktur

<html>
  <head>
    <title>This is the title.</title>
  </head>
  <body>
    <h1>This is a headline.</h1>

    This is the body.
    And this is more of the body.
  </body>
</html>
  • Knoten (nodes): Elemente, Attribute, Text, Kommentare
  • Children, Siblings, Parent, Ancestors

Wohlgeformtheit (well-formed)

Dokument erfüllt die XML-Syntaxregeln:

  • genau ein Wurzelelement
  • alle Elemente geschlossen (End-Tag bzw. "/>")
  • korrekte Verschachtelung der Elemente
  • ein Element darf nicht mehrere gleichnamige Attribute besitzen

=> Darum kümmert sich der Editor!

Gültigkeit (valid)

  • Dokument ist wohlgeformt
  • gehorcht benutzerdefinierten semantischen Regeln => Schema

Schema: Veranstaltung

  • dozent
    • vorname
    • nachname
    • grad
    • funktion
  • thema
  • zeit
  • modus (z.B. "öffentlich")
  • entgelt (z.B. "gratis")

Schema: Veranstaltung (2)

<veranstaltung>
  <dozent><nachname>Krug</nachname>, <grad>D.</grad>
  <vorname>W. T.</vorname>, <funktion>P. O.</funktion></dozent>,
  <thema>Geschichte d. alten Philosophie, n. s. Lehrbuche</thema>,
  <zeit>10 U. 4 T.</zeit> <modus>öffentlich</modus>.
</veranstaltung>
  • Text und Elemente gemischt ("mixed content")
  • Reihenfolge der Elemente beliebig

Schema: Textabsätze

</vv>
